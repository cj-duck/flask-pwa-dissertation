Hardware

Native applications often use the ability to access a device's hardware components. These can be used to enhance a user's experience but also require user permission. To see if a PWA offers sufficient features for your application you can check whatwebcando.today to see what features are available on different browsers and operating systems.

Camera

For users to be able to upload or take photos on your application you can use an input tag in your HTML: 


  <input type="file" accept="image/*" capture="camera" />

On mobile devices this will launch the camera application for a file to be uploaded. You can test it out below:



We can use javascript to make this look better though, assigning the action to any element. As an example we'll make a button for taking a picture: 


  <button id='photofile' class="btn btn-success"><i class="fas fa-camera"></i></button>
  <input type="file" id='photofile1' accept="image/*" capture="camera" class="btn" style="display:none"/>

And the javascript to go along: 


  $("#photofile").click(function () {
      $("#photofile1").trigger('click');
  });

Geolocation

To access a devices location we need to first ask for permission. Let's first create a button to ask for permission with: 


  <button id="askLocation" class="btn btn-success">Request location permission</button>

And then in our JavaScript, when the button is clicked we'll check if the browser supports location services and get the user's location. This will automatically prompt the user for location access permissions: 


  if ('geolocation' in navigator) {
    document.getElementById('askLocation').addEventListener('click', function () {
      navigator.geolocation.getCurrentPosition(function (location) {
        console.log(location);
      });
    });
  } else {
    console.log('Geolocation API not supported.');
  }
